## 1. 交易策略总体框架

### 策略理念

平台采用**规则驱动 + 风险优先**的投资决策框架：

```
优先级层次（从高到低）：
┌──────────────────────────────────────────────────────┐
│  L1 铁律层（FORCE_SELL_ALL）                          │
│  • R_STOP_LOSS_15：持仓亏损 ≥15% 无条件清仓           │
│  • R_TREND_STOP_MA20：趋势恶化，提前离场               │
│  • R_QDII_PREMIUM_13：QDII 极端溢价，强制清仓          │
├──────────────────────────────────────────────────────┤
│  L2 风控层（NO_BUY / PAUSE_BUY）                      │
│  • 溢价过高禁买、RSI 超买止盈离场                      │
│  • QDII 折价消失、套利窗口关闭                         │
├──────────────────────────────────────────────────────┤
│  L3 机会层（WATCH_BUY）                               │
│  • 折价机会、技术超卖、52周低位等买入信号               │
└──────────────────────────────────────────────────────┘
```

**核心逻辑：**

- **先风控后盈利**：L1 规则触发时，无论当前盈亏状态，必须执行止损
- **不对称策略**：买入门槛高（需要明确超卖/折价信号），卖出保护多（多个退出机制）
- **策略完整性**：每个品种均有明确的入场信号（L3）和出场信号（L1/L2），形成完整交易闭环

### 交易信号动作说明

| 动作 | 颜色 | 操作含义 | 触发后续 |
|------|------|---------|---------|
| FORCE_SELL_ALL | 🔴 | 立即全部清仓 | 回测中卖出全部持仓 |
| NO_BUY | 🟠 | 禁止新建仓，**若持仓则平仓** | 回测中触发卖出 |
| PAUSE_BUY | 🟡 | 暂停建仓，持仓可继续持有 | 回测中不触发卖出 |
| WATCH_BUY | 🔵 | 建仓信号，空仓时可建仓 | 回测中触发买入 |
| HOLD | 🟢 | 无操作，继续观察 | 回测中不操作 |

> **关键点：** NO_BUY（禁止买入）在回测引擎中也会触发持仓平仓。这是因为 RSI 超买、QDII 溢价消失等信号既表示"不要买入"，也代表"当前是止盈的合适时机"。

## 3. QDII ETF 折溢价策略

### 策略逻辑

QDII ETF（如 513310 纳斯达克ETF）在 A 股交易，其市价与海外资产净值（IOPV）之间存在折溢价。

- **折价（premium_pct < 0）**：市价低于净值，存在套利空间
- **溢价（premium_pct > 0）**：市价高于净值，市场溢价投机

**本策略利用折溢价均值回归特性：在深度折价时买入，在溢价消失时卖出。**

### 入场条件

触发规则：`R_DISCOUNT_OPPORTUNITY`

```
premium_pct < -1.5%   （折价超过 1.5%）
AND ma20_pct > -5%     （当前价格不处于强下跌趋势）
```

**为何设 -1.5% 门槛而非 0？**

- 折价 < -1.5% 才有足够的套利安全边际（考虑交易成本约 0.1%）
- 避免在小折价时频繁进出，减少摩擦成本
- 仅在明显低估时建仓，提升胜率

**为何要求 ma20_pct > -5%？**

- 当价格大幅跌破 MA20（-5%以上），说明整体趋势不佳
- 在强下跌趋势中的折价可能继续扩大，不是安全的入场时机

### 出场条件

| 触发规则 | 出场原因 |
|---------|---------|
| `R_QDII_PREMIUM_ZERO_EXIT`（premium > 1%） | 折价消失，套利窗口关闭，正常止盈 |
| `R_PREMIUM_3`（3% < premium ≤ 10%） | 中度溢价，市场热度过高 |
| `R_PREMIUM_10`（premium > 10%） | 高溢价，立即离场 |
| `R_QDII_PREMIUM_13`（premium > 13%） | 极端溢价，L1 强制清仓 |
| `R_STOP_LOSS_15`（pnl ≤ -15%） | 持仓亏损达铁律止损线 |
| `R_TREND_STOP_MA20`（ma20 < -5% 且 rsi < 45） | 趋势恶化，提前止损 |

### 交易示例

```
2024-03-01  价格 3.20，premium -2.1%，MA20 -1.5%
→ 触发 R_DISCOUNT_OPPORTUNITY (WATCH_BUY)
→ 建仓

2024-04-10  价格 3.38，premium +1.5%
→ 触发 R_QDII_PREMIUM_ZERO_EXIT (NO_BUY)
→ 止盈离场
→ 盈利 = (3.38 - 3.20) / 3.20 = +5.6%（扣除成本约 +5.4%）
```

### 策略特点

| 特点 | 说明 |
|------|------|
| 套利空间明确 | 入场折价 > 1.5%，出场溢价 > 1%，理论套利窗口 ≥ 2.5% |
| 与海外市场相关 | QDII 净值跟踪海外标的，国内 A 股走势影响溢价率 |
| 流动性风险 | 溢价大幅扩大时难以快速出场 |
| 市场关闭风险 | 海外市场收盘后净值不更新，折溢价可能失真 |

## 4. A 股技术指标策略

### 策略逻辑

A 股策略使用 **MA20 均线偏离** 和 **RSI 超买超卖** 两个技术指标：

- 超跌时买入（均线偏离或 RSI 极度超卖）
- 超买时卖出（RSI > 70）

### 入场条件

**信号 1：** 触发规则 `R_A_STOCK_MA20_DIP`

```
ma20_pct < -3%    （价格低于 MA20 超过 3%，超跌）
AND rsi14 < 40    （RSI 处于弱势区间，尚未形成超买）
```

**信号 2：** 触发规则 `R_A_STOCK_RSI_OVERSOLD`

```
rsi14 < 30        （RSI 极度超卖，历史上均值回归概率高）
```

**两个信号可单独触发**。RSI < 30 的条件更严格，但不要求 MA20 偏离，适合价格短期急跌但均线尚未明显背离的情形。

### 出场条件

| 触发规则 | 出场原因 |
|---------|---------|
| `R_A_STOCK_RSI_OVERBOUGHT`（rsi14 > 70） | 技术超买，止盈离场 |
| `R_STOP_LOSS_15`（pnl ≤ -15%） | 铁律止损 |
| `R_TREND_STOP_MA20`（ma20 < -5% 且 rsi < 45） | 趋势恶化，提前止损 |

### 关键阈值说明

| 参数 | 值 | 解读 |
|------|-----|------|
| MA20 偏离买入线 | -3% | 偏离 3% 表示短期超跌，均值回归概率较高 |
| RSI 超卖线 | 30 | 传统 Wilder RSI 超卖区域 |
| RSI 超买线 | 70 | 传统 Wilder RSI 超买区域 |
| RSI 弱势配合 | 40 | MA20 偏离买入时，RSI 须 < 40 确认弱势（而非强势回调） |

### 交易示例

```
A股 RSI 超跌买入 + RSI 超买卖出：

2024-02-15  RSI 28，MA20 -1.5%
→ 触发 R_A_STOCK_RSI_OVERSOLD
→ 建仓（RSI < 30 极度超卖）

2024-03-20  RSI 73，价格上涨 12%
→ 触发 R_A_STOCK_RSI_OVERBOUGHT
→ 止盈离场，盈利约 +11.8%（扣成本）
```

## 5. 港股 52 周低位策略

### 策略逻辑

港股市场流动性相对较低，估值往往长期处于低位。**52 周低位策略**利用这一特性，在股价接近年内低点时建仓，等待估值修复。

### 入场条件

触发规则：`R_HK_STOCK_WEEK52_LOW`

```
week52_low_pct > 0     （当前价格高于 52 周最低价，即股价不在历史极低位）
AND week52_low_pct ≤ 10%  （当前价格在 52 周最低价上方 10% 以内）
```

**为何 0 < week52_low_pct ≤ 10%？**

- `> 0`：排除当前即为年内最低点的情形（还在下跌中）
- `≤ 10%`：10% 以内表示仍处于历史低位区间，具有安全边际
- 历史数据显示，年内低位上方 10% 以内往往是价值洼地

### 出场条件

| 触发规则 | 出场原因 |
|---------|---------|
| `R_HK_STOCK_RSI_OVERBOUGHT`（rsi14 > 70） | RSI 超买，技术超买止盈 |
| `R_STOP_LOSS_15`（pnl ≤ -15%） | 铁律止损 |
| `R_TREND_STOP_MA20`（ma20 < -5% 且 rsi < 45） | 趋势恶化 |

### week52_low_pct 计算方法

```
week52_low_pct = (当日价格 / 过去252个交易日最低价 - 1) × 100
```

**注意：** 需要至少 252 个交易日的历史数据才能得到真实的 52 周低位。历史数据不足时，系统使用已有数据的最低价。

## 6. 美股止跌反弹策略

### 策略逻辑

美股市场受到全球资金流动影响，存在明显的快速下跌后反弹现象（V型反转）。平台针对美股提供两种买入信号：

1. **RSI 超卖信号**（持续下跌后的传统技术超卖）
2. **单日大跌反弹信号**（急跌后的快速反弹机会）

### 入场条件

**信号 1：** 触发规则 `R_US_STOCK_RSI_OVERSOLD`

```
rsi14 < 30    （RSI 超卖）
```

**信号 2：** 触发规则 `R_US_STOCK_DRAWDOWN_BOUNCE`

```
day_drawdown_pct < -5%    （当日单日跌幅超过 5%）
AND pnl_pct > -15%         （持仓总亏损尚未达到铁律止损线）
```

**单日大跌信号说明：**

- 单日跌幅 > 5% 通常是非理性抛售（财报超预期 miss、宏观事件冲击、流动性踩踏）
- 这类急跌在 1-5 个交易日内往往有较高概率出现反弹修复
- 要求总亏损 < 15%，确保不在已深度亏损时再追加仓位

### 出场条件

| 触发规则 | 出场原因 |
|---------|---------|
| `R_US_STOCK_RSI_OVERBOUGHT`（rsi14 > 70） | RSI 超买，止盈 |
| `R_STOP_LOSS_15`（pnl ≤ -15%） | 铁律止损 |
| `R_TREND_STOP_MA20`（ma20 < -5% 且 rsi < 45） | 趋势恶化 |

### 历史回测表现（样本数据）

## 7. 公募基金低估值策略

### 策略逻辑

触发规则：`R_FUND_PB_LOW`

```
pb_ratio > 0       （PB 有效值）
AND pb_ratio < 1.0  （市净率低于 1，低于账面价值）
```

## 8. 铁律止损规则（全品种）

### R_STOP_LOSS_15 — 绝对止损线

**触发条件：** 持仓亏损（相对入场价）≥ 15%

```
pnl_pct ≤ -15   （回测中：相对入场价的实时亏损）
```

**这是不可被覆盖的铁律规则（L1 最高优先级）。**

**重要说明：** 在回测中，`pnl_pct` 在持仓期间会动态更新为相对入场价的盈亏，而非 CSV 中原始的累计收益。这确保止损基于实际交易成本触发。

### R_TREND_STOP_MA20 — 趋势止损

**触发条件：** 价格趋势恶化

```
ma20_pct < -5%    （价格跌破 MA20 超过 5%，趋势明显向下）
AND rsi14 < 45    （RSI 处于中性偏弱区间，动能不足）
```

**与 R_STOP_LOSS_15 的区别：**

| | R_STOP_LOSS_15 | R_TREND_STOP_MA20 |
|---|---|---|
| 触发依据 | 持仓亏损金额 | 价格与均线关系 |
| 触发时机 | 亏损到达上限时 | 趋势恶化时（可能在达到 15% 亏损前触发） |
| 意义 | 防止巨额亏损 | 提前规避趋势性下跌 |

**两个规则都可以触发止损，哪个先达到条件就先触发。**

### R_QDII_PREMIUM_13 — QDII 极端溢价封杀

**触发条件：** QDII ETF 溢价超过 13%

溢价超过 13% 表示市场严重高估，历史数据显示此后往往快速回落，立即清仓是最优选择。

## 9. 回测绩效解读

### 关键指标含义

| 指标 | 计算方式 | 越高越好？ | 合格线 |
|------|---------|---------|--------|
| **总收益率** | (期末价值 - 初始资金) / 初始资金 × 100 | ✅ | > 5% |
| **最大回撤** | 历史峰值到谷值的最大跌幅 | ❌（越低越好） | < 12% |
| **胜率** | 盈利笔数 / 总成交笔数 | ✅ | > 55% |
| **Calmar 比率** | 年化收益率 / 最大回撤 | ✅ | > 1.0 |
| **盈亏比** | 平均盈利 / 平均亏损 | ✅ | > 1.5（可选） |
| **Alpha** | 策略收益 - 基准指数收益 | ✅ | 正值 |

### Calmar 比率解读

Calmar 比率 = 年化收益率 / 最大回撤

| Calmar | 含义 |
|--------|------|
| < 0.5 | 策略质量差，冒了很大风险但回报不足 |
| 0.5 ~ 1.0 | 一般，风险与收益基本匹配 |
| 1.0 ~ 2.0 | 良好，风险调整后收益可接受 |
| > 2.0 | 优秀，以较低回撤实现了较高收益 |

### 胜率与盈亏比的关系

胜率高不等于策略好，需要同时看盈亏比：

| 情形 | 胜率 | 盈亏比 | 期望值 |
|------|------|--------|--------|
| 良好策略 | 60% | 2.0 | +1.2 |
| 中性策略 | 50% | 1.5 | +0.75 |
| 差策略 | 70% | 0.5 | +0.35 |
| 亏损策略 | 80% | 0.2 | -0.16 |

> 期望值 = 胜率 × 平均盈利比率 - 败率 × 平均亏损比率

### 交易成本影响

样本数据各品种成本统计：

| 品种 | 单边成本（买入+卖出+双向滑点） |
|------|------|
| A 股 | 约 0.27%（0.03 + 0.13 + 0.05×2） |
| ETF / QDII | 约 0.13%（0.015×2 + 0.05×2） |
| 港股 / 美股 | 约 0.30%（0.1×2 + 0.05×2） |

交易越频繁，成本侵蚀越大。策略设计上通过提高入场门槛来减少无效交易。

## 10. 绩效门限与上线条件

### 上线前必须通过的绩效门限

在将策略应用于实盘前，必须使用至少 **500+ 个交易日**的历史数据进行回测，并通过以下所有门限：

| 指标 | 门限 | 当前样本表现 |
|------|------|------------|
| 总收益率 | ≥ 5% | QDII 44% / A股 18% / HK 30% / US 23% |
| 最大回撤 | ≤ 12% | QDII 11.5% / A股 4.6% / HK 9.9% / US 10.8% |
| Calmar 比率 | ≥ 1.0 | QDII 1.76 / A股 1.93 / HK 1.44 / US 1.03 |
| 胜率 | ≥ 55% | QDII 67% / A股 60% / HK 78% / US 67% |

### 如何运行门限检验

```bash
stock-ai-research validate \
  --symbol 513310 \
  --csv data/my_real_qdii_history.csv \
  --gate configs/performance_gate.json \
  --is-qdii
```

输出 `"passed": true` 方可上线该标的。

### 门限通过后的上线步骤

```
1. validate 通过 → 策略质量满足要求
   ↓
2. 模拟交易至少 20 个交易日 → 验证信号与系统运行正常
   ↓
3. 检查 exec-alert 级别为 green → 执行链路无异常
   ↓
4. 确认 live-gate 未暂停 → 系统状态正常
   ↓
5. 开始实盘，初期小仓位（建议 ≤ 5% 总资金）
```

## 11. 仓位管理原则

### 回测引擎默认设置

- **单次仓位：** 满仓（`max_position_pct=1.0`，即将全部可用资金投入）
- **持仓方式：** 单标的持仓，不同时持有多仓位

这是简化的全仓模型，适合策略验证。实盘中应根据组合管理需求调整。

### 实盘仓位建议

| 场景 | 建议单标的仓位上限 |
|------|----------------|
| 单品种集中策略 | ≤ 30% 总资金 |
| 多品种分散组合 | ≤ 15% 总资金（平台默认） |
| 高波动品种（美股/港股） | ≤ 10% 总资金 |

平台风控模块默认的 `max_position_ratio=0.15`（15%）。

### 加仓原则

当前策略**不支持分批加仓**（空仓时遇到信号才建仓，持仓中不会再次买入）。

如需分批建仓，需在实盘操作层面手动分批，并通过 `max_position_pct` 参数控制每次建仓比例：

```bash
# 分批建仓：第一批建仓 50%
stock-ai-research backtest \
  --symbol 513310 \
  --csv data/sample_history_qdii.csv \
  --max-position-pct 0.5 \
  --is-qdii
```

## 12. 交易操作流程

### 标准开仓操作

```
Step 1: 确认信号（每日收盘后或开盘前）
─────────────────────────────────────
运行 decision 命令，传入最新行情快照
确认信号为 WATCH_BUY 且规则合理

Step 2: 风险复核
─────────────────────────────────────
检查 live-gate 状态（必须未暂停）
检查仓位上限（当前标的持仓 ≤ 15%）
检查 ETF 溢价（ETF 类：premium ≤ 10%）

Step 3: 执行下单
─────────────────────────────────────
使用唯一 idempotency-key 提交订单
记录订单 ID

Step 4: 确认成交
─────────────────────────────────────
查询 order-status 确认成交状态
若 PARTIAL_FILLED，根据市场情况决定是否 complete
```

### 标准平仓操作

```
Step 1: 确认出场信号
─────────────────────────────────────
信号为 NO_BUY 或 FORCE_SELL_ALL
或人工判断市场条件需要止盈/止损

Step 2: 执行卖出
─────────────────────────────────────
提交 SELL 订单
使用新的 idempotency-key

Step 3: 更新记录
─────────────────────────────────────
确认成交后，更新 watchlist 中的 pnl_pct 字段
记录本次交易盈亏
```

### 紧急止损操作

当 FORCE_SELL_ALL 信号出现时：

```bash
# 1. 立即确认信号
stock-ai-research decision --symbol 513310 --fields '{...}' --is-qdii
# action == "FORCE_SELL_ALL"

# 2. 立即下卖单
stock-ai-research trade \
  --env live \
  --symbol 513310 \
  --side SELL \
  --quantity [全部持仓数量] \
  --price [当前市价] \
  --latest-fields '{"price":[当前市价]}' \
  --idempotency-key emergency-sell-513310-$(date +%Y%m%d%H%M%S) \
  --confirm-token LIVE_OK \
  --risk-token RISK_OK
```

## 13. 信号冲突处理原则

### 多规则同时触发

当多个规则同时触发时，**级别最高（L1 > L2 > L3）的规则决定动作**：

| 情形 | 结果 |
|------|------|
| R_STOP_LOSS_15 (L1) + R_DISCOUNT_OPPORTUNITY (L3) 同时触发 | FORCE_SELL_ALL（L1 胜） |
| R_QDII_PREMIUM_ZERO_EXIT (L2) + R_A_STOCK_RSI_OVERSOLD (L3) 同时触发 | NO_BUY（L2 胜） |
| R_A_STOCK_MA20_DIP (L3) + R_A_STOCK_RSI_OVERSOLD (L3) 同时触发 | WATCH_BUY（同级取第一个） |

**实操建议：** 即使决策卡片显示多个触发规则，也要重点关注 `action` 字段，它已经是最终优先级结果。

### 买入信号遇到趋势止损

当 WATCH_BUY 和 R_TREND_STOP_MA20 同时触发时（如 RSI < 30 且 MA20 < -5%）：

- 系统返回 FORCE_SELL_ALL（L1 R_TREND_STOP_MA20 胜过 L3 WATCH_BUY）
- **实操含义：** 空仓情况下，不要在趋势恶化时买入，等趋势止损条件消失后再等待入场信号

## 14. 市场异常情况处理

### 熔断情况（市场单日大幅下跌）

```bash
# 1. 检查持仓止损状态
stock-ai-research decision --symbol YOUR_SYMBOL --fields '{"price":[当前价],"ma20_pct":[MA20偏离],"rsi14":[RSI]}'

# 2. 若触发 FORCE_SELL_ALL，立即清仓
# 3. 清仓后暂停实盘，等待市场稳定
stock-ai-research live-gate --action pause --reason "市场熔断，人工观察"
```

### 节假日休市处理

- 节假日前无需特殊操作，平台不自动执行
- 节后第一个交易日更新 watchlist 行情数据后再运行决策

### 数据异常（价格错误/字段缺失）

**症状：** 决策返回异常结果（如 premium_pct 显示 0 但实际有折溢价）

**处理：**
1. 检查 watchlist 中 fields 数据是否正确更新
2. 不要基于异常数据执行交易
3. 数据来源核对后再运行

### 平台系统故障

若平台无法正常运行：

```bash
# 检查 live-gate 是否已自动暂停
stock-ai-research live-gate --action status

# 若未暂停，手动暂停
stock-ai-research live-gate --action pause --reason "系统故障，人工介入"
```

故障期间不执行任何实盘操作，等系统恢复并验证正常后再恢复。

## 15. 策略回顾与优化周期

### 每月策略检视清单

```
□ 运行批量回测，统计各标的绩效门限状态
□ 检查各规则的触发频率（是否与预期相符）
□ 评估近月实盘胜率与模拟回测的差异
□ 检视最大单笔亏损（是否有规则未及时触发的情形）
□ 审计 trade_events.jsonl 中是否有异常订单
□ 评估执行告警历史（长期 orange/red 的标的需审查）
```

### 规则参数调整触发条件

| 情形 | 建议动作 |
|------|---------|
| 胜率连续 3 月 < 45% | 检查买入门槛是否过低，考虑收紧入场条件 |
| 最大回撤突破门限 | 检查止损规则是否及时触发，可考虑降低 R_TREND_STOP_MA20 阈值 |
| 交易次数为 0（连续 2 周无信号） | 检查市场状态，评估门槛是否过高 |
| 盈亏比 < 1.0 | 入场信号质量差，需要检查数据准确性和规则逻辑 |

### 市场状态变化应对

当市场进入新的宏观阶段（如持续牛市、政策大幅调整），需要：

1. 用最新 1-2 年数据重新运行回测
2. 验证现有规则参数在新市场状态下的有效性
3. 必要时调整阈值后，重新运行 `validate` 验证门限

> **警告：** 不要基于短期（< 3 个月）表现调整规则参数，避免过拟合。

## 16. LLM 新闻辅助信号

### 功能定位

LLM 新闻分析是**L4 辅助层信号**，仅供交易员参考，不影响 L1-L3 规则引擎的买卖动作。

```
优先级层次（从高到低）：
┌──────────────────────────────────────────────────────┐
│  L1 铁律层（FORCE_SELL_ALL）                          │
├──────────────────────────────────────────────────────┤
│  L2 风控层（NO_BUY / PAUSE_BUY）                      │
├──────────────────────────────────────────────────────┤
│  L3 机会层（WATCH_BUY）                               │
├──────────────────────────────────────────────────────┤
│  L4 辅助层 ★NEW（新闻情感分析）                       │
│  • 仅展示，不产生交易动作                             │
│  • 辅助判断 WATCH_BUY 信号的确信度                    │
└──────────────────────────────────────────────────────┘
```

### 情感信号解读

| 情感 | 含义 | 对交易的参考意义 |
|------|------|---------------|
| 📈 **看涨（bullish）** | 新闻整体正面，市场情绪乐观 | 与 WATCH_BUY 信号共振时，可适当提高建仓信心 |
| 📉 **看跌（bearish）** | 新闻整体负面，存在利空因素 | WATCH_BUY 信号出现时保持谨慎，考虑缩小仓位 |
| ➡️ **中性（neutral）** | 消息面无明显方向 | 依赖技术规则信号做决策，新闻分析无额外参考价值 |

### 风险等级解读

| 风险等级 | 含义 | 操作建议 |
|---------|------|---------|
| 🟢 低（low） | 无明显风险事件 | 正常执行规则信号 |
| 🟡 中（medium） | 存在潜在风险因素 | 关注规则信号，控制仓位上限 |
| 🔴 高（high） | 存在重大风险事件（政策、财报、宏观冲击等） | 即使规则信号为 WATCH_BUY，也建议等待风险落地后再建仓 |

### 使用场景示例

**场景 1：规则信号与新闻信号共振**
```
规则信号：WATCH_BUY（RSI 超卖）
新闻情感：📈 看涨  风险等级：低
参考结论：信号可信度较高，可按正常仓位建仓
```

**场景 2：规则信号与新闻信号相悖**
```
规则信号：WATCH_BUY（52周低位）
新闻情感：📉 看跌  风险等级：高
参考结论：技术信号触发但消息面有压力，建议减半仓位或等待观察
```

**场景 3：HOLD 状态下的新闻预警**
```
规则信号：HOLD
新闻情感：📉 看跌  风险等级：高
参考结论：规则层暂无信号，但消息面风险上升。建议设置较紧的心理止损线
```

### 重要限制说明

1. **新闻覆盖范围有限**：仅美股（如 AAPL）和港股支持个股新闻。A股和 ETF 标的无法获取个股新闻，可使用「大盘」模式获取宏观情感参考。

2. **新闻分析存在滞后**：新闻通常反映已发生事件，而规则信号基于价格行为。极端行情中价格变动快于新闻更新。

3. **LLM 可能误判**：大模型在理解行业背景、中文语境时存在局限，情感判断不保证准确。建议交叉参考多条要点，而非只看情感标签。

4. **不构成投资建议**：新闻分析结果仅作为辅助参考工具，不代表平台对未来市场走势的预测。

## 17. 风险揭示

### 策略固有风险

1. **趋势风险**
   规则基于历史统计规律，在市场长期单边趋势（如熊市持续 2 年）下，即使触发了买入信号，价格也可能继续下跌。铁律止损 -15% 是最终保护线，但下跌到 -15% 本身仍是亏损。

2. **规则失效风险**
   RSI、MA20 等技术指标在不同市场环境下有效性不同。历史参数的有效性不代表未来同样有效。建议每季度验证规则的统计显著性。

3. **流动性风险**
   信号触发时可能无法按预期价格成交（尤其是港股小盘股、QDII ETF 极端溢价时）。实际成交价与信号触发价的偏差可能影响实际盈亏。

4. **数据质量风险**
   `fields` 中的行情数据质量直接影响决策准确性。数据延迟、错误或缺失会导致错误信号。务必使用可靠的数据源。

5. **成本侵蚀风险**
   高频交易会积累大量交易成本（A股单边约 0.16%，双边约 0.32%）。如果信号太频繁，成本会显著侵蚀收益。

6. **集中度风险**
   如将全部或大比例资金集中于单一品种，一旦规则失效或极端事件发生，损失将被放大。建议分散到多品种。

7. **系统延迟风险**
   晨报或批量监控的运行时间与实际交易时间之间可能存在价格变化，使决策基于过时数据。

### 本平台不保证

- ❌ 不保证未来实盘收益率
- ❌ 不保证规则在所有市场环境下有效
- ❌ 不保证样本数据回测结果代表真实未来表现
- ❌ 不构成任何投资建议

### 合理预期

| 指标 | 合理预期范围 | 说明 |
|------|------------|------|
| 年化收益率 | 8% ~ 25% | 取决于市场状态和品种选择 |
| 最大回撤 | 5% ~ 15% | 铁律止损保护，但仍可能达到止损线 |
| 胜率 | 55% ~ 70% | 高胜率高盈亏比策略的典型范围 |
| 有效交易频率 | 每标的每年 5~20 次 | 低频策略，减少成本侵蚀 |
