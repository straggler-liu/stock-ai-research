## 1. 系统架构总览

平台分为五层：

```
┌─────────────────────────────────────────────────────────────────┐
│  L5  接口层     CLI (main.py) / Python API                      │
├─────────────────────────────────────────────────────────────────┤
│  L4  协调层     orchestrator.py   monitoring.py   report.py     │
├──────────────────────────────────────┬──────────────────────────┤
│  L3  研究层     rules.py            │  执行层  trade_service.py │
│                backtest.py          │         execution.py      │
│                validation.py        │         risk.py           │
├──────────────────────────────────────┴──────────────────────────┤
│  L2  基础层     router.py  models.py  feishu_card.py            │
│                notifier.py  execution_alerts.py  live_gate.py   │
├─────────────────────────────────────────────────────────────────┤
│  L1  数据层     CSV历史文件  JSON配置  JSONL事件日志             │
└─────────────────────────────────────────────────────────────────┘
```

**核心原则：**

- **研究与执行解耦** — rules/backtest 不依赖 execution，可独立测试
- **零外部依赖** — 全部使用 Python 标准库，无需 pip install 第三方包
- **配置驱动** — 阈值在 JSON 配置中，不硬编码业务参数
- **幂等安全** — 订单通过 idempotency_key 防重复提交
- **可审计** — 所有交易事件写入 JSONL 日志，永不覆盖

### 完整规则表（default_rules()）

共 16 条规则，按优先级从高到低：

| 规则ID | 级别 | 动作 | 触发条件 | 适用品种 |
|--------|------|------|---------|---------|
| R_STOP_LOSS_15 | L1 | FORCE_SELL_ALL | pnl_pct ≤ -15% | 全品种 |
| R_TREND_STOP_MA20 | L1 | FORCE_SELL_ALL | ma20_pct < -5% **且** rsi14 < 45 | 全品种 |
| R_QDII_PREMIUM_13 | L1 | FORCE_SELL_ALL | premium_pct > 13% | QDII ETF |

| R_US_STOCK_RSI_OVERSOLD | L3 | WATCH_BUY | rsi14 < 30 | US_STOCK |
| R_US_STOCK_DRAWDOWN_BOUNCE | L3 | WATCH_BUY | day_drawdown_pct < -5% **且** pnl_pct > -15% | US_STOCK |
| R_FUND_PB_LOW | L3 | WATCH_BUY | 0 < pb_ratio < 1.0 | FUND |

### 评估算法

```python
def evaluate_rules(snapshot: MarketSnapshot, rules: list[Rule]) -> Decision:
    results = [rule.evaluate(snapshot) for rule in rules]
    triggered = [r for r in results if r.triggered]

    if not triggered:
        return Decision(..., action="HOLD")

    triggered.sort(key=lambda r: r.level)  # L1 < L2 < L3
    winner = triggered[0]                   # 最高优先级规则获胜

    return Decision(
        action=winner.action,
        reasons=[r.reason for r in triggered],
        triggered_rule_ids=[r.rule_id for r in triggered],
    )
```

### 动作说明

| 动作 | 颜色 | 含义 |
|------|------|------|
| FORCE_SELL_ALL | 🔴 | 立即清仓，回测中触发卖出 |
| NO_BUY | 🟠 | 禁止买入，**回测中若持仓则触发卖出** |
| PAUSE_BUY | 🟡 | 暂停买入，回测中不触发卖出 |
| WATCH_BUY | 🔵 | 关注买入机会，**回测中触发买入** |
| HOLD | 🟢 | 无动作 |

> **重要**：NO_BUY 在回测引擎中会触发持仓卖出，因此 RSI>70 等止盈规则设为 NO_BUY 可实现完整的买入-卖出循环。

### pnl_pct 字段语义

回测执行时，快照中的 `pnl_pct` 会被引擎**动态覆盖**为基于实际入场价的持仓盈亏百分比：

```python
# 在 backtest.py 中，持仓期间自动覆盖
if quantity > 0 and entry_price > 0:
    fields["pnl_pct"] = (price - entry_price) / entry_price * 100
```

这确保 R_STOP_LOSS_15 基于**实际建仓价**触发，而非 CSV 中以第1天价格为基准的累计收益。

各品种样本 CSV 的列：

| 品种 | CSV 文件 | 列 |
|------|---------|-----|
| QDII ETF | sample_history_qdii.csv | date, price, benchmark_price, premium_pct, pnl_pct, rsi14, ma20_pct |
| A 股 | sample_history_a_stock.csv | date, price, benchmark_price, premium_pct, pnl_pct, ma20_pct, rsi14 |
| 港股 | sample_history_hk_stock.csv | date, price, benchmark_price, premium_pct, pnl_pct, week52_low_pct, rsi14, ma20_pct |
| 美股 | sample_history_us_stock.csv | date, price, benchmark_price, premium_pct, pnl_pct, rsi14, day_drawdown_pct, ma20_pct |

### 交易成本（硬编码）

| 品种 | 买入费率 | 卖出费率 | 滑点（单边） |
|------|---------|---------|------------|
| A_STOCK | 0.03% | 0.13%（含印花税） | 0.05% |
| CN_ETF | 0.015% | 0.015% | 0.05% |
| QDII_ETF | 0.015% | 0.015% | 0.05% |
| FUND / HK_STOCK / US_STOCK / REITS | 0.1% | 0.1% | 0.05% |

### 回测逻辑

```
for 每个交易日:
    1. 解析 CSV 行 → fields 字典
    2. 若持仓中：覆盖 fields["pnl_pct"] = 基于入场价的实时盈亏
    3. 构建 MarketSnapshot → 评估规则 → Decision
    4. 若 WATCH_BUY 且空仓且剩余行数 ≥ 2：
       - 买入 = cash × max_position_pct
       - 实际买价 = price × (1 + buy_rate + slippage)
    5. 若 FORCE_SELL_ALL 或 NO_BUY 且有持仓：
       - 实际卖价 = price × (1 - sell_rate - slippage)
       - 记录盈亏，更新 wins/losses
    6. 更新净值，计算最大回撤
最终：计算 CAGR / Calmar / Alpha 等指标
```

### 报告字段

```python
@dataclass
class BacktestReport:
    symbol: str
    trades: int                   # 买入笔数（不含卖出）
    total_return_pct: float       # 总收益率 %
    max_drawdown_pct: float       # 最大回撤 %
    win_rate_pct: float           # 胜率 %
    cagr_pct: float               # 年化收益率 %
    calmar: float                 # CAGR / 最大回撤
    benchmark_return_pct: float   # 基准收益率 %
    alpha_pct: float              # 超额收益 %
    total_cost_pct: float         # 总交易成本占初始资金 %
    profit_loss_ratio: float      # 平均盈利 / 平均亏损（无亏损时=999.0）
    max_consecutive_losses: int   # 最大连续亏损笔数
```

## 8. 风控模块

文件：[src/stock_ai_research/risk.py](../src/stock_ai_research/risk.py)

```python
def pretrade_risk_check(
    order: TradeOrder,
    latest_fields: dict,
    account_total_value: float,
    existing_position_value: float,
    max_position_ratio: float = 0.15,
    instrument_type: InstrumentType | None = None,
) -> RiskCheckResult
```

**检查项：**

1. **价格字段存在性** — `latest_fields` 中必须包含 `"price"`
2. **订单有效性** — quantity > 0 且 price > 0
3. **仓位上限** — `(existing_position_value + order_value) / account_total_value ≤ max_position_ratio`（默认 15%）
4. **ETF 溢价** — premium_pct ≤ 10%（仅 CN_ETF / QDII_ETF）

**失败代码：**

| 代码 | 含义 |
|------|------|
| `missing_price` | latest_fields 缺少 price |
| `invalid_order_value` | 订单 quantity 或 price 为 0 |
| `position_limit_exceeded` | 超过单标的仓位上限 |
| `premium_over_10_no_buy` | ETF 溢价超 10% |

```python
    idempotency_key: str,
    confirm_token: str = "",
    risk_token: str = "",
) -> tuple[str, Fill]
```

执行流程：
1. 检查幂等键 → 若已存在，直接返回缓存结果
2. 生成 `order_id`（16位 UUID hex）
3. 记录 SUBMITTED 事件
4. 调用 BrokerAdapter.place_order
5. 记录 FILLED / PARTIAL_FILLED 事件
6. 保存幂等映射
7. 返回 `(order_id, fill)`

#### cancel_order / complete_order

```python
def cancel_order(self, order_id: str) -> dict
def complete_order(self, order_id: str) -> dict
```

## 19. 样本数据生成

文件：[scripts/generate_sample_data.py](../scripts/generate_sample_data.py)

```bash
python scripts/generate_sample_data.py
```

**设计原则：**

- 使用 `simulate_cyclic_prices()`：多轮 bull→dip→recover 循环，每轮回撤 ≤ 12%
- QDII premium 通过 `simulate_premium_correlated()` 与价格动量正相关
- 美股使用 `simulate_us_prices_with_shocks()`：主趋势上涨，偶发单日大跌（-5%~-7.5%）触发 R_US_STOCK_DRAWDOWN_BOUNCE
- 所有 CSV 固定随机种子（SEED=42），结果可复现

**参数调整（影响策略触发频率）：**

| 参数 | 位置 | 效果 |
|------|------|------|
| `dip_mu` | simulate_cyclic_prices | 回调深度（越负越深） |
| `dip_sigma` | simulate_cyclic_prices | 回调波动率 |
| `bull_mu` | simulate_cyclic_prices | 牛市日均涨幅 |
| `price_sensitivity` | simulate_premium_correlated | premium 对价格的跟随强度 |
| `shock_probability` | simulate_us_prices_with_shocks | 美股单日大跌概率 |

```python
class MyBroker(BrokerAdapter):
    def place_order(self, order: TradeOrder) -> Fill:
        # 调用券商 API
        response = broker_api.submit(

            side=order.side,

            price=order.price,

        return Fill(
            symbol=order.symbol,
```
