## 1. 快速开始

```bash
# 克隆项目
git clone <repository_url>
cd stock-ai-research

# 安装（纯 Python 标准库，无外部依赖）
pip install -e .

# 验证安装
pytest -q
# 应显示：59 passed

# 生成样本数据
python scripts/generate_sample_data.py

# 运行一次决策
stock-ai-research decision \
  --symbol 513310 \
  --fields '{"price":3.792,"premium_pct":19.95,"pnl_pct":8.2}' \
  --is-qdii
```

## 2. 安装与环境

### 系统要求

| 项目 | 要求 |
|------|------|
| Python | 3.10 及以上 |
| 操作系统 | Windows / macOS / Linux |
| 外部依赖 | 无（仅使用标准库） |
| 磁盘空间 | 约 50 MB（含历史数据） |

### 安装步骤

```bash
# 1. 创建虚拟环境（推荐）
python -m venv .venv
source .venv/bin/activate     # Linux/macOS
# 或：.venv\Scripts\activate  # Windows

# 2. 安装平台
pip install -e .

# 3. 生成样本历史数据
python scripts/generate_sample_data.py

# 4. 验证所有测试通过
pytest -q

# 5. 查看 CLI 帮助
stock-ai-research --help
```

### 目录初始化

首次运行时，`data/` 目录下的 JSON 文件（orders.json、paper_ledger.json 等）会自动创建。如需手动初始化：

```bash
mkdir -p data
echo '{}' > data/idempotency.json
echo '{}' > data/orders.json
echo '{"positions": []}' > data/paper_ledger.json
echo '{"paused": false, "reason": "", "updated_at": ""}' > data/live_gate_state.json
```

## 4. 命令行使用

### 4.1 单标的决策

```bash
stock-ai-research decision \
  --symbol 513310 \
  --fields '{"price":3.792,"premium_pct":19.95,"pnl_pct":8.2}' \
  --is-qdii
```

**参数说明：**

| 参数 | 说明 | 示例 |
|------|------|------|
| `--symbol` | 证券代码 | `513310` |
| `--fields` | JSON 格式行情快照 | `'{"price":3.5,"rsi14":28}'` |
| `--is-qdii` | 标记为 QDII ETF | （flag，无值） |
| `--is-fund` | 标记为公募基金 | （flag，无值） |

**输出示例：**

```json
{
  "action": "FORCE_SELL_ALL",
  "status": "🔴",
  "reasons": ["QDII ETF溢价超过13%，触发封杀清仓。"],
  "triggered_rule_ids": ["R_QDII_PREMIUM_13"]
}
```

### 4.2 历史回测

```bash
stock-ai-research backtest \
  --symbol 513310 \
  --csv data/sample_history_qdii.csv \
  --is-qdii
```

**可选参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--initial-cash` | 100000 | 初始资金（元） |
| `--max-position-pct` | 1.0 | 单次最大仓位比例（1.0=满仓） |

**输出示例：**

```json
{
  "symbol": "513310",
  "trades": 36,
  "total_return_pct": 44.44,
  "max_drawdown_pct": 11.5,
  "win_rate_pct": 66.67,
  "calmar": 1.76
}
```

### 4.3 绩效门限验证

```bash
stock-ai-research validate \
  --symbol 513310 \
  --csv data/sample_history_qdii.csv \
  --gate configs/performance_gate.json \
  --is-qdii
```

**输出示例：**

```json
{
  "passed": true,
  "failures": [],
  "report": { "trades": 36, "total_return_pct": 44.44, ... }
}
```

### 4.4 批量监控

```bash
stock-ai-research batch \
  --config configs/sample_watchlist.json
```

带绩效门限验证：

```bash
stock-ai-research batch \
  --config configs/sample_watchlist.json \
  --enforce-gate \
  --gate configs/performance_gate.json
```

带飞书推送：

```bash
stock-ai-research batch \
  --config configs/sample_watchlist.json \
  --webhook "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" \
  --enforce-gate
```

### 4.5 晨报

```bash
stock-ai-research brief \
  --config configs/sample_watchlist.json \
  --webhook "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" \
  --enforce-gate \
  --market-news "今日A股高开，半导体板块领涨"
```

### 4.6 订单管理

```bash
# 查询订单
stock-ai-research order-status --order-id a1b2c3d4e5f6a7b8

# 撤销订单
stock-ai-research cancel --order-id a1b2c3d4e5f6a7b8

# 完成部分成交
stock-ai-research complete --order-id a1b2c3d4e5f6a7b8
```

### 4.7 实盘熔断

```bash
# 查询状态
stock-ai-research live-gate --action status

# 暂停实盘
stock-ai-research live-gate --action pause --reason "系统维护"

# 恢复实盘
stock-ai-research live-gate --action resume
```

## 5. watchlist 配置管理

文件：`configs/sample_watchlist.json`

### 结构说明

```json
[
  {
    "symbol": "513310",
    "is_qdii": true,
    "history_csv": "data/sample_history_qdii.csv",
    "fields": {
      "price": 3.792,
      "iopv": 3.1614,
      "premium_pct": 19.95,
      "pnl_pct": 8.2,
      "rsi14": 65.0,
      "ma20_pct": 2.1
    }
  },
  {
    "symbol": "600519",
    "history_csv": "data/sample_history_a_stock.csv",
    "fields": {
      "price": 1530.0,
      "pnl_pct": -4.5,
      "ma20_pct": -4.0,
      "rsi14": 32.0
    }
  }
]
```

### 各品种必填字段

| 品种 | 必填 fields | 可选 fields |
|------|------------|------------|
| QDII ETF | price, premium_pct | iopv, rsi14, ma20_pct, pnl_pct |
| A 股 | price, ma20_pct, rsi14 | pnl_pct |
| 港股 | price, week52_low_pct, rsi14 | ma20_pct, pnl_pct |
| 美股 | price, rsi14 | day_drawdown_pct, ma20_pct, pnl_pct |
| 基金 | price, pb_ratio | pnl_pct |

### 每日更新 fields

`fields` 中的数据需每日更新为当日收盘后的最新值。建议通过脚本从数据源获取后，更新 watchlist JSON 文件。

> **重要**：`pnl_pct` 字段在 watchlist 中填写的是您实际持仓的盈亏百分比（相对于您的建仓成本），而非 CSV 历史数据中的累计收益。监控场景下这个字段用于触发止损规则。

## 6. 历史数据准备

### 使用样本数据（快速验证）

```bash
python scripts/generate_sample_data.py
```

生成 4 个 CSV 文件（各 520 行约 2 年历史），可直接用于回测和绩效验证。

### 使用真实历史数据

#### CSV 格式要求

**QDII ETF：**

```
date,price,benchmark_price,premium_pct,pnl_pct,rsi14,ma20_pct
2023-01-03,3.1250,100.5500,-1.52,-0.01,48.32,-0.15
2023-01-04,3.1480,100.7200,-1.23,0.74,52.10,0.25
...
```

**A 股：**

```
date,price,benchmark_price,premium_pct,pnl_pct,ma20_pct,rsi14
2023-01-03,1620.50,3080.50,0.0,-0.01,-0.32,51.2
...
```

**港股：**

```
date,price,benchmark_price,premium_pct,pnl_pct,week52_low_pct,rsi14,ma20_pct
2023-01-03,305.60,18250.00,0.0,1.87,2.10,54.3,0.85
...
```

**美股：**

```
date,price,benchmark_price,premium_pct,pnl_pct,rsi14,day_drawdown_pct,ma20_pct
2023-01-03,182.50,4020.00,0.0,1.39,52.1,0.0,0.85
...
```

#### 字段计算说明

| 字段 | 计算方法 |
|------|---------|
| `premium_pct` | (ETF市价 / IOPV - 1) × 100 |
| `pnl_pct` | 可设为 0（回测引擎会基于实际入场价动态覆盖） |
| `rsi14` | Wilder RSI，14日，初始期填 50 |
| `ma20_pct` | (当日收盘价 / 20日均线 - 1) × 100 |
| `week52_low_pct` | (当日价格 / 过去252日最低价 - 1) × 100 |
| `day_drawdown_pct` | (当日收盘价 / 前日收盘价 - 1) × 100 |
| `benchmark_price` | 基准指数价格（如沪深300、恒生、标普500） |

#### 数据质量检查

```bash
# 检查数据行数
wc -l data/my_history.csv

# 检查日期升序
head -5 data/my_history.csv
tail -5 data/my_history.csv

# 运行回测验证
stock-ai-research backtest \
  --symbol MY_SYMBOL \
  --csv data/my_history.csv
```

## 7. 绩效门限配置

文件：`configs/performance_gate.json`

### 默认配置

```json
{
  "min_total_return_pct": 5.0,
  "max_drawdown_pct": 12.0,
  "min_calmar": 1.0,
  "min_win_rate_pct": 55.0
}
```

### 调整建议

| 场景 | 建议配置 |
|------|---------|
| 保守型（降低上线门槛） | max_drawdown_pct: 15.0, min_calmar: 0.8 |
| 激进型（提高策略质量要求） | min_total_return_pct: 10.0, min_win_rate_pct: 60.0 |
| 添加交易频率检验 | 追加 "min_trades": 8 |
| 添加盈亏比检验 | 追加 "min_profit_loss_ratio": 1.5 |
| 添加连亏检验 | 追加 "max_consecutive_losses": 4 |

> **注意**：修改门限后，需重新运行 `validate` 命令验证所有标的是否仍通过。

## 8. 执行告警配置

文件：`configs/execution_alerts.json`

### 告警分级说明

| 级别 | 含义 | 平台动作 |
|------|------|---------|
| green | 执行质量正常 | 无 |
| yellow | 轻微异常，观察 | 无 |
| orange | 执行问题，需关注 | 飞书推送告警 |
| red | 严重异常 | 飞书推送 + 自动暂停实盘 |

### 配置说明

```json
{
  "default": {
    "max_reject_ratio": 0.2,    // 全局最大拒单率（超过 → 告警）
    "max_cancel_ratio": 0.4,    // 全局最大撤单率
    "min_events": 5             // 最小事件数（数据不足时不触发）
  },
  "by_env": {
    "live": {                    // 实盘环境单独设更严格阈值
      "max_reject_ratio": 0.1,
      "max_cancel_ratio": 0.2,
      "min_events": 3
    }
  },
  "by_symbol": {                 // 特定标的的专属阈值
    "513310": {
      "max_cancel_ratio": 0.35,
      "min_events": 4
    }
  },
  "windows": {
    "7d": { "days": 7, "max_reject_ratio": 0.15, ... },
    "30d": { "days": 30, "max_reject_ratio": 0.2, ... }
  }
}
```

### 新增标的专属告警

```json
"by_symbol": {
  "513310": { "max_cancel_ratio": 0.35 },
  "600519": { "max_reject_ratio": 0.05, "min_events": 3 }
}
```

```json
  "order_id": "a1b2c3d4e5f6a7b8",
```

```bash

  --env paper \
  --symbol 513310 \
  --side SELL \
  --quantity 1000 \
  --price 3.35 \
  --latest-fields '{"price":3.35,"premium_pct":1.5}' \
```

## 10. 实盘交易操作

> ⚠️ **实盘交易为高风险操作，必须完成以下前置检查后方可执行。**

### 前置检查清单

```bash
# 1. 确认实盘熔断未触发
stock-ai-research live-gate --action status
# 返回：{"paused": false, ...} 才可继续

# 2. 确认绩效门限通过
stock-ai-research validate \
  --symbol 513310 \
  --csv data/sample_history_qdii.csv \
  --gate configs/performance_gate.json \
  --is-qdii
# 返回：{"passed": true} 才可继续

# 3. 确认决策信号
stock-ai-research decision \
  --symbol 513310 \
  --fields '{"price":3.20,"premium_pct":-2.1}' \
  --is-qdii
# 确认 action == "WATCH_BUY" 才执行买入
```

### 提交实盘订单

```bash
stock-ai-research trade \
  --env live \
  --symbol 513310 \
  --side BUY \
  --quantity 1000 \
  --price 3.20 \
  --latest-fields '{"price":3.20,"premium_pct":-2.1}' \
  --idempotency-key live-buy-513310-20260329 \
  --confirm-token LIVE_OK \
  --risk-token RISK_OK
```

**双 Token 说明：**

| Token | 用途 |
|-------|------|
| `--confirm-token LIVE_OK` | 操作员确认：我知道这是实盘操作 |
| `--risk-token RISK_OK` | 风控确认：风控检查已通过 |

缺少任一 Token，实盘订单将被拒绝。

### 实盘风控自动检查

提交实盘订单时，系统自动执行：

1. 价格字段存在性验证
2. 订单金额有效性
3. 单标的仓位占比 ≤ 15%（默认）
4. ETF 溢价 ≤ 10%（QDII/CN ETF）

如任一检查失败，返回 `risk_check.passed = false` 并拒绝下单。

## 11. 晨报自动化

### 手动运行晨报

```bash
stock-ai-research brief \
  --config configs/sample_watchlist.json \
  --webhook "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" \
  --enforce-gate \
  --market-news "今日市场消息：美联储维持利率不变"
```

### 自动化（定时任务）

**Linux/macOS（cron）：**

```bash
# 每个工作日 9:00 发送晨报
crontab -e
# 添加：
0 9 * * 1-5 cd /path/to/stock-ai-research && \
  python -c "
import json
from stock_ai_research.monitoring import run_morning_brief
brief = run_morning_brief(
    'configs/sample_watchlist.json',
    webhook_url='https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN',
    enforce_gate=True,
    market_news='自动晨报'
)
print(json.dumps(brief, ensure_ascii=False, indent=2))
" >> logs/morning_brief.log 2>&1
```

**Windows（任务计划程序）：**

创建 `run_brief.bat`：

```batch
@echo off
cd /d D:\stock-ai-research
.venv\Scripts\python.exe -m stock_ai_research.main brief ^
  --config configs/sample_watchlist.json ^
  --webhook "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" ^
  --enforce-gate
```

在任务计划程序中设置每个工作日 9:00 运行。

### 晨报卡片内容

发送至飞书的晨报包含：

1. **市场总览** — 今日监控标的数量，各动作类型统计（🔴止损/🟠禁买/🟡暂停/🔵关注/🟢持有）
2. **行动清单** — 需要处理的标的，含具体动作和原因
3. **策略绩效摘要**（enforce_gate=True 时）— 回测平均收益、回撤、Calmar
4. **执行质量状态** — green/orange/red 分级
5. **市场资讯** — 传入的 market_news 内容

## 12. 批量回测与报告

### 批量回测所有标的

```bash
stock-ai-research report \
  --config configs/sample_watchlist.json \
  --out-json data/backtest_report.json \
  --out-csv data/backtest_report.csv
```

### 查看报告

**JSON 报告**（程序化读取）：

```json
{
  "summary": {
    "count": 4,
    "avg_total_return_pct": 29.13,
    "avg_max_drawdown_pct": 9.18,
    "avg_calmar": 1.55
  },
  "reports": [...]
}
```

**CSV 报告**（Excel/Spreadsheet）：

```
symbol,trades,total_return_pct,max_drawdown_pct,win_rate_pct,cagr_pct,calmar,...
513310,36,44.44,11.5,66.67,21.3,1.76,...
```

### 定期回测建议

每周至少运行一次批量回测，关注：
- 总收益率是否持续 > 5%
- 最大回撤是否仍 < 12%
- Calmar 是否仍 > 1.0
- 胜率是否仍 > 55%

若任一指标退化，需检查规则参数是否需要调整。

## 14. 实盘熔断控制

### 自动熔断

当执行告警级别达到 **red** 时，系统会在告警报告中标记 `"pause_live": true`。**注意：自动熔断需要运营人员手动确认并执行：**

```bash
# 执行告警检测到 red 级别后，手动暂停
stock-ai-research live-gate --action pause --reason "exec_alert_red"
```

### 手动熔断

```bash
# 暂停实盘（如系统维护、市场异常）
stock-ai-research live-gate --action pause --reason "市场熔断，人工介入"

# 查看状态
stock-ai-research live-gate --action status
# {"paused": true, "reason": "市场熔断，人工介入", "updated_at": "2026-03-29T10:00:00+00:00"}
```

### 恢复实盘

熔断原因解除后：

```bash
# 先确认执行质量正常
stock-ai-research exec-alert --events-log data/trade_events.jsonl \
  --alert-config configs/execution_alerts.json
# 确认 level == "green" 后恢复

# 恢复实盘
stock-ai-research live-gate --action resume
```

### 熔断期间的行为

- `run_watchlist(live_gate_paused=True)` 调用时，所有标的 `live_allowed=False`
- 飞书推送不会发出
- 模拟交易不受影响

## 15. LLM 新闻分析配置

### 功能说明

平台支持通过大模型（LLM）自动抓取和分析市场新闻，作为**辅助参考信号**展示在桌面应用「新闻分析」Tab 中。该功能不影响规则引擎的买卖决策动作。

支持 4 种大模型提供方，可在界面实时切换：

| 提供方 | 模型 | API Key 获取 |
|--------|------|------------|
| OpenAI | gpt-4o-mini | platform.openai.com |
| Claude (Anthropic) | claude-haiku-4-5-20251001 | console.anthropic.com |
| Gemini (Google) | gemini-2.0-flash | aistudio.google.com |
| DeepSeek | deepseek-chat | platform.deepseek.com |

### 配置 API Key

**方法一：通过桌面应用（推荐）**

1. 打开 `python app.py`，切换到「⚙️ 配置管理」Tab
2. 找到「LLM 新闻分析配置」区域
3. 选择默认提供方，填入对应 API Key（输入时显示 `*` 隐藏）
4. 点击「保存 LLM 配置」

**方法二：直接编辑配置文件**

编辑 `configs/llm_settings.json`（若不存在则手动创建）：

```json
{
  "active_provider": "openai",
  "openai_key": "sk-...",
  "claude_key": "sk-ant-...",
  "gemini_key": "AIza...",
  "deepseek_key": "sk-..."
}
```

> ⚠️ **安全提示**：`configs/llm_settings.json` 已加入 `.gitignore`，不会被提交到 Git 仓库。请勿手动将其添加到版本控制中。

### 使用新闻分析

1. 打开 `python app.py`，切换到「📰 新闻分析」Tab
2. 在「标的」下拉框选择要分析的标的（或「大盘（市场整体）」）
3. 在「LLM 提供方」下拉框选择要使用的模型
4. 点击「开始分析」按钮
5. 等待约 5-15 秒后，结果区域显示：
   - **市场情感**：📈 看涨 / 📉 看跌 / ➡️ 中性（绿/红/灰底色）
   - **风险等级**：低 / 中 / 高
   - **核心要点**：3 条关键信息
   - **一句话摘要**
   - **相关资讯**：新闻列表（双击可打开原文链接）

### 品种覆盖说明

| 品种 | 是否有新闻 | 说明 |
|------|----------|------|
| 美股（AAPL、MSFT 等） | ✅ | Yahoo Finance RSS |
| 港股（00700 等） | ✅ | Yahoo Finance RSS（`700.HK`） |
| A股（600519 等） | ❌ | Yahoo 无覆盖，返回「No articles found」 |
| QDII ETF（513310 等） | ❌ | 同上 |
| 大盘（市场整体） | ✅ | Yahoo S&P 500 RSS，备用 Reuters |

> **提示**：A股和 ETF 标的选择「大盘（市场整体）」可获取宏观市场情感分析作为参考。

### API Key 未配置时的行为

如果 API Key 为空，点击「开始分析」后，状态栏会显示：

```
分析失败：LLM returned empty response
```

请先在「配置管理」Tab 填入对应提供方的 API Key。

## 18. 常见问题排查

### Q1：决策一直返回 HOLD，没有信号

**原因：** fields 中缺少规则所需字段，或字段值不在触发阈值范围内。

**排查：**

```bash
# 检查字段是否完整
stock-ai-research decision \
  --symbol 600519 \
  --fields '{"price":1530,"ma20_pct":-4.0,"rsi14":32.0,"pnl_pct":-4.5}'
# ma20_pct < -3 AND rsi14 < 40 → 应触发 WATCH_BUY
```

### Q2：回测 trades=0，没有交易

**原因：** 规则信号未触发（如 QDII 的 premium_pct 始终大于 -1.5%）。

**排查：**

```bash
python -c "
import csv
rows = list(csv.DictReader(open('data/sample_history_qdii.csv', encoding='utf-8')))
disc = [r for r in rows if float(r['premium_pct']) < -1.5]
print('Deep discount days:', len(disc))
"
```

若为 0，说明样本数据需要重新生成：

```bash
python scripts/generate_sample_data.py
```

### Q3：绩效门限验证失败

**常见失败原因及处理：**

| 失败项 | 可能原因 | 建议 |
|--------|---------|------|
| `max_drawdown_pct>12.0` | 样本数据熊市段过深 | 重新生成样本数据 |
| `win_rate_pct<55.0` | 买入信号与趋势止损冲突 | 检查是否存在 R_TREND_STOP_MA20 同时触发 |
| `calmar<1.0` | 回撤过大或收益过低 | 综合调整回测参数 |
| `total_return_pct<5.0` | 历史数据整体下跌 | 检查数据质量 |

### Q4：飞书推送失败

**排查步骤：**

```bash
# 测试网络连通性
curl -X POST "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"msg_type":"text","content":{"text":"test"}}'

# 检查 webhook URL 格式
# 正确：https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxx
# 错误：http:// 或缺少 hook 路径
```

### Q5：实盘订单被拒绝

查看拒绝原因：

```bash
stock-ai-research order-status --order-id YOUR_ORDER_ID
# 查看 payload 中的失败原因
```

常见原因：
- `missing_price` — latest-fields 中未传 price
- `position_limit_exceeded` — 单标的仓位超 15%
- `premium_over_10_no_buy` — ETF 溢价过高

### Q6：回测 pnl_pct 与预期不符

回测引擎会动态覆盖 `pnl_pct`，使其基于实际入场价计算，而非 CSV 中的累计值。这是设计行为，确保止损规则按入场成本触发，而非按历史第一天价格。
